# Reflection (Assignment 01)

This project made one thing extremely clear: prompt engineering quality directly controls product quality. At first, the temptation was to write short generic persona prompts and rely on model intelligence. But that quickly produced bland responses where all three personas sounded similar. The biggest improvement came from treating each system prompt like a product spec: explicit identity, communication style, few-shot examples, constraints, and output shape.

What worked best was separating “what the persona knows” from “how the persona speaks.” For example, all three personas discuss career growth, DSA, and interview prep, but they should still feel different. I used Anshuman’s style as execution-heavy and skills-first, Abhimanyu’s style as strategic and systems-oriented, and Kshitij’s style as concept-first and teaching-oriented. Once these distinctions were embedded through few-shot examples, the model’s voice became much more stable turn-to-turn. The follow-up question rule also helped keep the interaction conversational instead of becoming one-sided advice dumps.

The GIGO principle showed up repeatedly. Weak input (“be helpful and friendly”) gave weak output (generic motivational text). Better input (clear values, realistic examples, explicit constraints) produced significantly better response quality and persona consistency. I also learned that constraints are not optional; they are essential. Without constraints, models can overclaim, hallucinate details, or offer unrealistic guarantees like “you will definitely get a 30 LPA job in 3 months.” With constraints, responses stayed grounded and professional.

If I had more time, I would improve two areas. First, I would run systematic prompt evaluations with a fixed test suite of user queries and score each persona on tone consistency, actionability, and factual safety. Second, I would add analytics to track where users drop off in conversations and refine suggestion chips accordingly. Overall, the project reinforced that prompt engineering is not “writing one clever instruction.” It is iterative product design where context quality, behavioral constraints, and output formatting all combine to create a reliable user experience.
